Phase 2d reference vector — t10_vm_single_fp_smoke_rn.bin
=========================================================

Spec attestation
----------------
T10 attests Rust `dispatch_instruction` byte-for-byte parity with
the v2 RandomX fork's `executeInstruction` for the FP-arithmetic
opcode classes under round-to-nearest (MXCSR mode 0). Four single-
opcode probes:

  - FADD_R  : opcode 124, dst=0, src=0
              — fdst += a[src]
  - FMUL_R  : opcode 172, dst=1, src=2
              — edst *= a[src]
  - FDIV_M  : opcode 204, dst=2, src=3, mod=0 (L2 mask), imm32=0x040
              — edst /= maskRegisterExponentMantissa(load[spAddr])
  - FSQRT_R : opcode 208, dst=3
              — edst = sqrt(edst)

Each probe runs against a fresh canonical NativeRegisterFile and
emits a 256-byte T4-layout snapshot.

A T10 divergence under RN isolates a bug in the corresponding FP
dispatch arm independent of rounding mode (T11-T14 cover RD/RU/RZ).

Per `docs/design/RANDOMX_V2_PHASE2D_PLAN.md` §6.2.

Inputs / scratchpad
-------------------
Same canonical NativeRegisterFile + scratchpad pattern as T9
(documented in gen.cpp's CANONICAL_* tables). MXCSR set via
`rx_set_rounding_mode(0)` before each probe.

Output layout
-------------
4 × 256-byte NativeRegisterFile snapshots, one per probe in the
order listed above. Total: 1024 bytes.
