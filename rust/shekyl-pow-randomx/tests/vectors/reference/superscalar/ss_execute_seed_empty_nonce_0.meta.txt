SuperscalarHash execution vector — ss_execute_seed_empty_nonce_0.bin
====================================================================

Spec attestation
----------------
Cross-checks shekyl-pow-randomx's `superscalar::execute_superscalar`
(spec §7.3 step 5: `SuperscalarHash[i](r0..r7)`) against the v2
RandomX fork's `randomx::executeSuperscalar`. Layer B vector paired
with Layer A program `ss_program_seed_empty_nonce_0.bin` in the F4
structured 3-vector decomposition per
`docs/design/RANDOMX_V2_PHASE2B_PLAN.md` §5.4.

The Rust-side test loads the serialized program from the Layer A
`.bin`, deserializes via the wire format, then executes against
the fixed input below — decoupling generation parity (Layer A)
from execution parity (Layer B). A divergence on a Layer B vector
while the corresponding Layer A vector passes attributes the
failure to `execute_superscalar` (instruction dispatch / register
state update / reciprocal computation), not to program generation.

Inputs (hardcoded in `_generator/gen.cpp`)
------------------------------------------
program = Layer A vector `ss_program_seed_empty_nonce_0.bin`
          (regenerated locally for self-containment)
input_r = [0, 1, 2, 3, 4, 5, 6, 7]  — uint64_t each

The fixed `r=[0..8]` input minimizes Layer B entropy: the input is
the simplest non-degenerate permutation of the 8 register slots, so
any byte-diff between expected and actual is attributable to the
execution path, not to a complex input shape.

Output layout
-------------
64 bytes = `r[8]` after `executeSuperscalar(r, prog)`, each register
written as little-endian `u64`. Matches the RandomX spec §7.3 step 5
register output convention.
