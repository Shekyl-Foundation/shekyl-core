SuperscalarHash combined attestation vector — ss_combined_seed_shekyl_nonce_0.bin
=================================================================================

Spec attestation
----------------
End-to-end attestation tuple per
`docs/design/RANDOMX_V2_PHASE2B_PLAN.md` §5.4: a single canonical
"this is what a correct SuperscalarHash port produces for these
inputs" reference. Downstream consumers — other Shekyl repos,
alternative SS-hash ports, audit reviewers without the Rust port
source — can verify their own implementation against this single
vector without needing to parse the wire-format-specific Layer A
serialization.

This vector is byte-identical to `ss_execute_seed_shekyl_nonce_0.bin`
by construction. The two files are tested through different code
paths on the Rust side: the Layer B test loads the serialized
program from a Layer A `.bin` and runs `execute_superscalar`
against it (decoupling generation from execution parity); the
combined test regenerates the program locally via
`generate_superscalar` and then executes (testing the full
generate → execute pipeline without intermediate serialization,
mirroring how downstream consumers would actually use the SS-hash).

Inputs (hardcoded in `_generator/gen.cpp`)
------------------------------------------
seed    = b"shekyl-ss-test"  — 14 ASCII bytes
nonce   = 0
input_r = [0, 1, 2, 3, 4, 5, 6, 7]  — uint64_t each

The full pipeline: `Blake2Generator(seed, nonce)` →
`generateSuperscalar(prog, gen)` →
`executeSuperscalar(input_r, prog)` →
emit `input_r` as 8 × `u64` LE.

Output layout
-------------
64 bytes = post-execution `r[8]` as 8 × `u64` little-endian.
