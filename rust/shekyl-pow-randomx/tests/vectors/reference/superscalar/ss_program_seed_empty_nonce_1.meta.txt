SuperscalarHash program serialization vector — ss_program_seed_empty_nonce_1.bin
================================================================================

Spec attestation
----------------
Layer A nonce-mixing isolation vector in the F4 structured 3-vector
decomposition per `docs/design/RANDOMX_V2_PHASE2B_PLAN.md` §5.4.
Shares the (empty) seed with `ss_program_seed_empty_nonce_0.bin`
and differs only in nonce. A test that asserts this byte-for-byte
against `generate_superscalar(Blake2Generator::new(&[], 1))`
isolates the Blake2Generator's nonce-mixing path: a divergence
here while the empty-seed/nonce=0 vector passes attributes the
failure to nonce handling specifically (not seed initialization,
not the downstream port-assign / instruction-selection pipeline).

Inputs (hardcoded in `_generator/gen.cpp`)
------------------------------------------
seed   = (nullptr, size=0)  — same as vector 1 by intent
nonce  = 1                  — differs from vector 1 by exactly the
                              nonce-mixing surface

documented at `_generator/README.md` "Wire format" + on the Rust
side at `src/superscalar.rs`'s `superscalar_vector_format` module
rustdoc. Total bytes for this vector: 3640 = 8 header + 8 * 454
instructions.
