SuperscalarHash execution vector — ss_execute_seed_shekyl_nonce_0.bin
=====================================================================

Spec attestation
----------------
Cross-checks shekyl-pow-randomx's `superscalar::execute_superscalar`
(spec §7.3 step 5) against the v2 RandomX fork's
`randomx::executeSuperscalar`. Layer B vector paired with Layer A
program `ss_program_seed_shekyl_nonce_0.bin` (the seed-derivation
isolation program). See `ss_execute_seed_empty_nonce_0.meta.txt`
for the Layer B vector rationale; this vector exercises the same
execution path against the third Layer A program.

This vector is byte-identical to `ss_combined_seed_shekyl_nonce_0.bin`
by construction. The two files are tested through different code
paths on the Rust side: the Layer B test (`ss_execute_*`) loads the
serialized program from `.bin` and runs `execute_superscalar` against
it; the combined test (`ss_combined_*`) regenerates the program
locally via `generate_superscalar` and then executes. A divergence
between the two on the Rust side would indicate the wire-format
round-trip is lossy, which the Layer A tests already catch
independently.

program = Layer A vector `ss_program_seed_shekyl_nonce_0.bin`

written as little-endian `u64`.
