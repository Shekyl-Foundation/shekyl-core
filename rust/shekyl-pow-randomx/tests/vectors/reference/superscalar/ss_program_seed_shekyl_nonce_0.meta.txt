SuperscalarHash program serialization vector — ss_program_seed_shekyl_nonce_0.bin
=================================================================================

Spec attestation
----------------
Layer A seed-derivation isolation vector in the F4 structured
3-vector decomposition per `docs/design/RANDOMX_V2_PHASE2B_PLAN.md`
§5.4. Shares `nonce=0` with `ss_program_seed_empty_nonce_0.bin`
and differs only in the seed. A test that asserts this byte-for-byte
against `generate_superscalar(Blake2Generator::new(b"shekyl-ss-test", 0))`
isolates the Blake2Generator's seed-initialization and seed-byte-
consumption paths: a divergence here while the empty-seed vector
passes attributes the failure to seed handling specifically.

Inputs (hardcoded in `_generator/gen.cpp`)
------------------------------------------
seed   = b"shekyl-ss-test"  — 14 ASCII bytes; arbitrary but distinct
                              from the empty seed, exercising both
                              the seed-pointer-non-null branch and a
                              non-trivial Blake2Generator data fill
nonce  = 0                   — same as vector 1 by intent

documented at `_generator/README.md` "Wire format" + on the Rust
side at `src/superscalar.rs`'s `superscalar_vector_format` module
rustdoc. Total bytes for this vector: 3536 = 8 header + 8 * 441
instructions.
